# 個人グローバル設定

全プロジェクト共通。プロジェクトの CLAUDE.md が優先。

## 言語
- 応答は日本語（常体，簡潔）．
- 日本語の句読点は `，` と `．` を使う（`、` と `。` は使わない）．応答，コメント，コミットメッセージ，PR すべてに適用．
- コメント，コミットメッセージ，PR は日本語．識別子は英語．
- 絵文字は明示的に頼まれた時のみ．

## VCS: jj（git ではない）
- 編集前に `jj st`．`@` がこれからの作業用の新しい change になっているか確認．`@` に予期しない変更があれば停止して確認．新しい change がなければ `jj new` してから．
- `jj describe` は Conventional Commits（例: `feat: ユーザー検索を追加`）．
- `jj git push` は確認不要．`--force` や push 済み change の書き換えは確認必須．
- pre-commit hook は絶対にスキップしない．失敗したら根本原因を直す．

## シェル
- シェル経由で叩く時はモダン CLI を優先: `grep` より `rg`，`find` より `fd`，`cat` より `bat`（Read が使えない場面で）．
- 破壊的操作（`rm -rf`，一括削除，未知の change への `jj abandon` など）は実行前に確認．
- `$HOME`（`/Users/citrus`）の外には触らない．システムパスは事前承認が必要．
- 30秒超のコマンド（ビルド，テスト，サーバ）は `run_in_background` で起動．
- グローバルインストール（`npm i -g`，`brew install` など）は無断でしない．
